Detailed Description

Implementation of queue data structure for Merkle tree construction.

This file contains the implementation of a queue_t data structure specifically designed for use in the Merkle tree construction process. It includes functions for initialization, memory management, and basic queue_t operations. The implementation uses a sentinel node design for efficient queue operations.

Removes up to n elements from the front of the queue.

Dequeues the minimum of the requested count and available elements. Allocates an array to hold the pointers and populates it by calling pop_queue() repeatedly. Updates the count parameter with the actual number of elements returned.

Parameters
qPointer to the queue.
countPointer to requested count (input) and actual count (output).
resultPointer to store the allocated array of element pointers.
Returns
QUEUE_OK on success, error code on failure.
Note
Caller is responsible for freeing the returned array and managing the element data according to their ownership model.
If memory allocation fails, *count is set to 0.

Frees all memory associated with the queue, including its elements.

Iterates through all nodes in the queue (excluding the sentinel) and frees them. If a deallocator function is provided, it's called for each element's data. Finally frees the sentinel node and queue structure.
